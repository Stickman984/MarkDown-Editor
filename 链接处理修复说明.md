# 链接处理修复说明

## 问题描述
用户点击 Markdown 文档中的链接时，仍然使用 Chrome 浏览器打开，而不是按照我们设计的智能处理方式。

## 问题原因
`tkhtmlview` 库有自己的链接点击处理机制。在 `HTMLScrolledText` 类中，有一个 `on_link_click` 属性用于处理链接点击事件。我们之前的做法（尝试重写 `set_html` 和绑定 `hyperlink-` 标签）无法正确拦截父类的默认行为。

## 解决方案
直接覆盖 `HTMLScrolledText` 的 `on_link_click` 方法：

```python
class CustomHTMLScrolledText(HTMLScrolledText):
    def __init__(self, *args, viewer=None, **kwargs):
        self.viewer = viewer
        super().__init__(*args, **kwargs)
        
        # 覆盖父类的链接点击处理
        self.on_link_click = self._custom_link_click
    
    def _custom_link_click(self, url):
        """自定义链接点击处理"""
        if self.viewer:
            self.viewer.handle_link_click(url)
        else:
            webbrowser.open(url)
```
