# 测试代码块缩进问题

## 情况1：代码块前有4个空格（不显示）

    ```
    [ro.lmk.swap_free_low_percentage]: [15]
    ```

## 情况2：代码块前无空格（正常显示）

```
[ro.lmk.swap_free_low_percentage]: [15]
```

## 情况3：代码块前有2个空格（应该显示）

  ```
  [ro.lmk.swap_free_low_percentage]: [15]
  ```

## 情况4：列表中的代码块（缩进必须）

- 列表项1
  
  ```
  [ro.lmk.swap_free_low_percentage]: [15]
  ```

- 列表项2

## 原因说明

在 Markdown 中：
- **4个空格或1个Tab的缩进** = 传统的缩进式代码块
- 当你的 ``` 前面有4个空格时，Markdown会认为这整个内容（包括```符号）都是缩进代码块
- 这会导致 ``` 本身被当作代码内容显示，而不是作为 fenced code block 的边界
- 因此内部的内容就不会被正确解析

## 解决方案

1. **不要在代码块前加4个或更多空格**（除非在列表中）
2. 如果在列表中，确保缩进正确对齐列表内容
3. 使用0-3个空格的缩进是安全的
