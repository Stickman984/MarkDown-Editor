# 最终解决方案：显示 [key]: [value] 格式内容

## 问题
`[key]: [value]` 会被 Markdown 当作链接引用定义，导致<font color="#ff0000">不显示</font>。

---

## ✅ 推荐方案（按优先级）

### 方案1：使用无缩进代码块（最佳）
```
[ro.lmk.swap_free_low_percentage]: [15]
[ro.lmk.swap_util_max]: [85]
[ro.lmk.thrashing_threshold]: [30]
```

**用法**：代码块 ``` 前面不要有任何空格

---

### 方案2：使用行内代码
单个配置项：`[ro.lmk.swap_free_low_percentage]: [15]`

---

### 方案3：使用4个空格缩进代码块
    ```
    [ro.lmk.swap_free_low_percentage]: [15]
    ```

注意：这会让 ``` 本身也显示出来，效果如下：
    ```
    [ro.lmk.swap_free_low_percentage]: [15]
    ```

---

### 方案4：列表中的特殊处理

**错误做法**（不显示）：
- 配置项：
  
  ```
  [ro.lmk.swap_free_low_percentage]: [15]
  ```

**正确做法1**（使用行内代码）：
- 配置项：`[ro.lmk.swap_free_low_percentage]: [15]`

**正确做法2**（使用HTML代码块）：
- 配置项：
  
  <pre><code>[ro.lmk.swap_free_low_percentage]: [15]</code></pre>

**正确做法3**（添加语言标识）：
- 配置项：
  
  ```properties
  [ro.lmk.swap_free_low_percentage]: [15]
  ```

让我测试一下方案3（添加语言标识）是否有效：

- 测试properties语言
  
  ```properties
  [ro.lmk.swap_free_low_percentage]: [15]
  ```

- 测试text语言
  
  ```text
  [ro.lmk.swap_free_low_percentage]: [15]
  ```

- 测试ini语言
  
  ```ini
  [ro.lmk.swap_free_low_percentage]: [15]
  ```

---

### 方案5：使用表格
| 属性 | 值 |
|------|-----|
| `ro.lmk.swap_free_low_percentage` | `15` |
| `ro.lmk.swap_util_max` | `85` |

---

## ❌ 不要使用的方案

1. ❌ 1-3个空格的缩进代码块
2. ❌ 直接写 `[key]: [value]`（无任何格式）
3. ❌ 列表中使用2个空格缩进的普通代码块
4. ❌ 引用块中使用普通代码块

---

## 快速参考

| 场景 | 推荐方案 |
|------|---------|
| 单个配置项 | 行内代码：`[key]: [value]` |
| 多个配置项 | 无缩进代码块 |
| 在列表中 | 行内代码 或 添加语言标识的代码块 |
| 在表格中 | 用反引号包裹 |

---

## 原理说明

Markdown 会在以下情况把内容当作**链接引用定义**：
- `[标识符]: URL` 格式
- 在段落级别被解析时
- 1-3个空格缩进不会阻止解析

只有这些情况能阻止解析：
- ✅ 在 fenced code block 中（0个空格缩进）
- ✅ 在缩进代码块中（4个空格）
- ✅ 添加了语言标识的 fenced code block（可能）
- ✅ 使用转义字符
- ✅ 使用行内代码
