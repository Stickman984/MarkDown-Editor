# Markdown 方括号显示问题总结

## 核心问题

`[key]: [value]` 这种格式会被 Markdown 误认为是**链接引用定义**，导致不显示。

---

## ✅ 能正常显示的写法

### 1. 行内代码（最推荐）
`[ro.lmk.swap_free_low_percentage]: [15]`

### 2. 代码块（无缩进）
```
[ro.lmk.swap_free_low_percentage]: [15]
[ro.lmk.swap_util_max]: [85]
[ro.lmk.thrashing_threshold]: [30]
```

### 3. 使用转义字符
\[ro.lmk.swap_free_low_percentage\]: \[15\]

### 4. 表格（内容用代码格式）
| 属性名 | 值 |
|--------|-----|
| `ro.lmk.swap_free_low_percentage` | `15` |
| `ro.lmk.swap_util_max` | `85` |
| `ro.lmk.thrashing_threshold` | `30` |

### 5. 列表中的代码块（正确缩进）
- 配置项1：
  
  ```
  [ro.lmk.swap_free_low_percentage]: [15]
  ```

- 配置项2：
  
  ```
  [ro.lmk.swap_util_max]: [85]
  ```

---

## ❌ 不能显示的写法

### 1. 直接写（被当作链接引用定义）
[ro.lmk.swap_free_low_percentage]: [15]

### 2. 代码块前有4个空格（被当作缩进代码块）
    ```
    [ro.lmk.swap_free_low_percentage]: [15]
    ```

### 3. 表格中直接用方括号（被当作链接引用）
| 属性 | 值 |
|------|-----|
| [ro.lmk.swap_free_low_percentage] | [15] |

---

## 💡 最佳实践建议

1. **显示配置项/属性值**：使用代码块或行内代码
2. **在列表中**：确保代码块缩进2个空格，不要4个
3. **在表格中**：使用反引号包裹内容
4. **避免**：直接写 `[key]: [value]` 格式

---

## 原理说明

### Markdown 链接引用定义语法
```markdown
[标识符]: URL "可选标题"
[google]: https://www.google.com "Google"
```

### 缩进代码块规则
- **4个空格或1个Tab** = 缩进代码块
- 会导致 ``` 符号本身变成代码内容
- 推荐使用 **0-3个空格**
